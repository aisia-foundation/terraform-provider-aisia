# Provider AISIA

Le provider **aisia** est le provider Terraform officiel de la plateforme [AISIA](https://aisia.fr).
Il permet de gérer organisations, utilisateurs, clés API, clés providers LLM, guardrails, webhooks,
revendeurs et modèles locaux en **Infrastructure-as-Code**.

Source : [github.com/aisia-foundation/terraform-provider-aisia](https://github.com/aisia-foundation/terraform-provider-aisia)
Registry : [registry.terraform.io/providers/aisia-foundation/aisia](https://registry.terraform.io/providers/aisia-foundation/aisia/latest/docs)

## Identité produit AISIA

AISIA est le **chef d'orchestre IA local-first** : une requête entre, le meilleur modèle (local ou cloud)
exécute, la réponse sort traçable et gouvernée.

**Fonction première** : orchestrer chaque requête en **local-first** via `BanditRouter` — pas un proxy LLM.

| vs proxy LLM | AISIA |
|--------------|-------|
| 1 provider fixe | **88** providers + **58** modèles locaux actifs |
| Stateless | Qdrant + audit AI Act + multi-tenant |
| SaaS opaque | Déployable Swarm/K8s — **v6.12.36** LIVE |

| Variable | Description |
|----------|-------------|
| `AISIA_TOKEN` | Clé API admin `ak_live_…` (Bearer) — **sensible** |
| `AISIA_ENDPOINT` | URL API (défaut `https://api.aisia.fr`) |

```terraform
provider "aisia" {
  # token via AISIA_TOKEN (recommandé)
  # endpoint = "https://api.aisia.fr"
}
```

## Example Usage

```terraform
terraform {
  required_providers {
    aisia = {
      source  = "aisia-foundation/aisia"
      version = "~> 6.12"
    }
  }
}

provider "aisia" {
  endpoint = "https://api.aisia.fr"
  token    = var.aisia_token
}

variable "aisia_token" {
  type      = string
  sensitive = true
}
```
